As I genuinly suck at math it took me quite a while to grasp what fellow developers meant explaining formulas for responsive sizing of webpage elements.

Initially I used a graph I had created on GeoGebra.org which suited me well for a while, ERS linear equations. A simple XY-graph with a few sample lines depicting equations I used in my webpages.

Although both equations yield the same result, a general CSS rule like:
 font-size: calc(mx + b)
e.g. font-size: calc(0.625vmin + 0.75rem)

is much easier to maintain than one like:
 font-size: calc((y2 - y1) / (x2 - x1) * (x - x1) + y1)
e.g. font-size: calc((14 + (20 - 14) / (1280 - 320) * (100vmin - 320)) * 1px), should you get this code running at all as using vmin like that yields an error in the calc().

calculate result using linear equation

'slope intercept' form
y = mx + b
 =>
y = (y2 - y1) / (x2 - x1) + (y1 - (y2 - y1) / (x2 - x1) * x1)
'point slope' form
y - y1 = m(x - x1) => y = y1 + m(x - x1)
 =>
y = y1 + (y2 - y1) / (x2 - x1) * (x - x1)
As the 'point slope' form is shortest, linearToCalc  uses that equation to create the CSS calc()

for points

p1: (x1,y1)
x1 - minimum viewport value  
y1 - value at minimum viewport
p2: (x2,y2)
x2 - maximum viewport value  
y2 - value at maximum viewport

where

y: equation result
rise - how far up the line
m: (y2 - y1) / (x2 - x1)
slope - steepness of the line (change in Y / change in X) or (rise / run)
x: 100vmin
run - how far along the line (fixed 100 vmin, vh, vw or vmax)
b: y1 - m * x1
Y-intercept - where the line crosses the Y-axis (value at viewport size = 0)
y-scale:
viewport size
y-scale:
element size
